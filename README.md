<details>
<summary><b>🔷 RAG (Retrieval-Augmented Generation) — Click to Expand</b></summary>
<br/>

Advanced RAG concepts I implement:

```
📄 Document Ingestion
      ↓
🔪 Intelligent Chunking (semantic, recursive, hierarchical)
      ↓
🔢 Embedding Generation (OpenAI, HuggingFace, Cohere)
      ↓
🗄️ Vector Storage (Pinecone, ChromaDB, FAISS, Weaviate)
      ↓
🔍 Hybrid Search (Dense + Sparse / BM25 + Embeddings)
      ↓
📊 Re-Ranking (Cohere Rerank, Cross-Encoder)
      ↓
🧠 LLM Synthesis with Context
      ↓
✅ Grounded, Hallucination-Free Response
```

- **Multi-Query RAG** — generates multiple query variants for broader retrieval
- **Self-RAG** — model decides when and what to retrieve
- **Agentic RAG** — agents that can update, query, and reason over knowledge bases
- **Parent-Child Chunking** — retrieves child chunks, returns parent context

</details>

<details>
<summary><b>🔷 Transformer Architecture (Built from Scratch) — Click to Expand</b></summary>

I built a complete Transformer in PyTorch with **zero pre-trained weights**:

```python
# Core components I implemented from pure math:
✅ Multi-Head Self-Attention

✅ Encoder Stack
✅ Decoder Stack with Cross-Attention
✅ Causal Masking for autoregressive generation
✅ Training loop with label smoothing & learning rate warmup
```

This isn't a tutorial exercise — it's a demonstration of mathematical ownership over how modern AI models think.
